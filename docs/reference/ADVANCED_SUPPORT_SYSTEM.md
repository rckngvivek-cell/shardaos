# PART 2: AI CHATBOT SYSTEM (Tier 0)

## 2.1 Chatbot Architecture

```
┌────────────────────────────────────────────────────────────────┐
│                     USER MESSAGE INPUT                          │
│               "How do I enter marks offline?"                   │
└────────────────────┬───────────────────────────────────────────┘
                     ↓
┌────────────────────────────────────────────────────────────────┐
│              INTENT RECOGNITION ENGINE (NLU)                    │
│  • Claude or GPT-4 for natural language understanding           │
│  • Detects intent: ["how-to", "troubleshooting", "billing"]    │
│  • Extracts entities: [module: "exams", action: "offline"]     │
└────────────────────┬───────────────────────────────────────────┘
                     ↓
         ┌─────────────────────────────┐
         │   KNOWLEDGE BASE LOOKUP      │
         │   (Vector DB + Full-text)    │
         └────────┬────────────────────┘
                  ↓
    ┌─────────────────────────────────────┐
    │  Found answer in KB (confidence>80%)?│
    └─────────────────────────────────────┘
         YES ↓                    ↓ NO
         ↓                    ┌─────────────────────┐
    ┌─────────────────┐      │ GENERATE RESPONSE   │
    │  RETURN ARTICLE │      │ USING LLM           │
    │  WITH LINK & FAQs      │ (Chain-of-Thought)  │
    └────────┬────────┘      └────────┬────────────┘
             ↓                        ↓
      ┌──────────────────────────────────────┐
      │   CAN CHATBOT RESOLVE? (Binary)      │
      └──────────────────────────────────────┘
        YES ↓                      ↓ NO
        ↓                   ┌──────────────────────┐
    RESPOND WITH SOLUTION  │  ESCALATE TO SUPPORT │
    + Follow-up options    │  Tier 1 (Community)  │
    + Rating (thumbs up)   │  or Tier 2 (Agent)   │
                           └──────────────────────┘
```

## 5.2 Incident Response (P0 Critical)

```
WHEN: Data corruption, security breach, or mass outage detected

TEAM ASSEMBLED (within 15 mins):
├─ Incident Commander (rotation)
├─ 2-3 Engineering leads
├─ Senior support agent (customer liaison)
├─ DevOps engineer
└─ Product manager (visibility)

COMMAND CENTER:
├─ War room (Google Meet + Slack channel)
├─ Shared incident document (roles, timeline, actions)
├─ Status updates every 15 mins
└─ Customer communication every 30 mins

RESOLUTION TIMELINE:
├─ T+5 mins: Incident acknowledged to customer
├─ T+15 mins: Root cause identified
├─ T+45 mins: Interim workaround provided
├─ T+2 hours: Full fix deployed
├─ T+4 hours: Post-incident review

COMMUNICATION:
├─ Email: All affected school principals
├─ SMS: Critical alert (if data corruption)
├─ In-app: Banner notification
├─ Status page: Incident report + timeline
└─ Slack: Real-time updates to customers (Enterprise tier)
```
